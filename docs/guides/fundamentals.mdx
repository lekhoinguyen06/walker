## Components

### Server

The server is responsible for handling incoming requests and MAP from clients, processing them, and sending back ACTION. 

### Client

The client is responsible for sending MAP to the server, receiving ACTION from the server, and executing those actions to interact with the user application.

## Elements

### Action

Action, a JSON object, is a command sent from the server to the client to interact with the user application.

### Flow

Flow represents what command the client accepts and executes. It allows developers to declare what their application can do.

### Item

Item represents the individual elements that can be interacted with.

### Map

Map, a JSON object, is a command sent from the client to the server to describe the current landscape of the user application. For websites, this includes the DOM structure, element attributes, and other relevant information.


## Modes

### HTML 

This is to minimal config "no-code" solution to use Walker client in your web application. The library will handle crawling, caching, etc of your HTML elements efficiently to ensure minimal token usage without compromising walking experience.

Recommended for hobbyist or team without dedicated Agent engineers.

### Engineered

This is the option for developers to explicitly declared Items, Actions, and Flows for your Walker client applications.

Recommended for project with member dedicated to engineering custom tailored walking experience.
