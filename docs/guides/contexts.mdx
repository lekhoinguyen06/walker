---
title: "Contexts"
---

These are the places where you walking agent can get context from.

## Front 

Embedded in an HTML element as JSON in the `walker-data` attribute, you can provide component-level context on how to use your website.

You should declare Front context for necessary guidelines and descriptions. Users will benefit if they use browser personal agents that run on their machines, such as OpenClaw. If these personal agents adhere to these guidelines, they will consume your website appropriately.

## Back

For every walker component on your website, if you use Concierge or build your own Walker server, you can declare additional backend-level context.

You should declare Back context for privilege information and agent-engineered prompts. Privilege information, such as the latest promotions and product descriptions, should be stored in Back context. Concierge helps you manage the agent's Back knowledge base, try it out.

## Hook

The Walker library allows you to use hooks to provide custom context for your agent.

When you need customization, you can declare hooks. For example, a database hook allows your agent to have transactional awareness. Concierge provides a platform to observe, authenticate, control access, prevent information leaks, ensure governance, and more.
